# План реализации багов и фичей (v2)

На основе ответов на уточняющие вопросы, сформирован актуальный план работ по пет-проекту MOznoDS.

### 1. Исправление вёрстки: Поле ввода и окно созвона (#17)
- **Приоритет**: Высокий
- **Статус**: Готов к реализации
- **Что делаем**: 
  - Закрепляем окно созвона снизу.
  - Делаем его сворачиваемым так, чтобы оно **не перекрывало** поле ввода сообщения.
  - Для мобильных устройств — адаптивный дизайн (скрытие панелей, оптимизация пространства).

### 2. Баг с никами и отображением участников (#9)
- **Приоритет**: Высокий
- **Статус**: Готов к реализации
- **Что делаем**: 
  - Исправляем логику маппинга в `SignalingConsumer`.
  - Устраняем проблему, когда вместо ников отображается "User" или "You" некорректно для других участников.

### 3. Подсветка говорящего (#1)
- **Приоритет**: Средний
- **Статус**: Готов к реализации
- **Что делаем**: 
  - Реализуем на фронтенде анализ громкости через `AudioContext` + `AnalyserNode`.
  - Добавляем **зеленую рамку** вокруг аватара/ника говорящего участника.

### 4. Уведомления и обновление списка комнат (#2)
- **Приоритет**: Средний
- **Статус**: Готов к реализации
- **Что делаем**: 
  - Реализуем real-time обновление списка комнат через WebSocket при добавлении пользователя в новую комнату.

### 5. Административные функции (#7, #15)
- **Приоритет**: Средний
- **Что делаем**: 
  - Визуально выделяем админа (владельца комнаты) в списке участников.
  - Добавляем админу кнопку "попросить включить микрофон".

### 6. Улучшения WebRTC и Аудио (#8, #14)
- **Приоритет**: Средний
- **Что делаем**: 
  - Исправляем проблему с отображением видео (проверка SDP/ICE).
  - Добавляем локальную регулировку громкости каждого участника.

### 7. Мелкие правки (#6, #16)
- **Приоритет**: Низкий
- **Что делаем**: 
  - Mobile-first редизайн.
  - Исследование проблемы отображения эмодзи с iOS на Windows (шрифты).
